Complete Query Syntax Reference

Field Queries

SyntaxDescription
field:valueExact match
field:"phrase"Phrase match
field:wild*Wildcard (LIKE)
field:/regex/Regex match (~)

Boolean Operators

SyntaxDescription
AND, &&Conjunction
OR, ||Disjunction
NOT, !Negation
+termRequired (pass-through)
-termProhibited (NOT)

Range Queries

SyntaxDescription
[a TO b]Inclusive (BETWEEN)
{a TO b}Exclusive (> AND <)
[a TO b}Mixed (>= AND <)
>=, >, <=, <Comparison operators
[a TO *]Unbounded range

Advanced Queries

SyntaxDescription
term~NFuzzy (edit distance N)
"phrase"~NProximity (within N words)
term^NBoost (relevance weight)
_exists_:fieldField existence (IS NOT NULL)

Grouping

SyntaxDescription
(a OR b)Parentheses grouping
field:(a OR b)Field group

Operator Precedence

Lowest to highest:

  1. OR (lowest)
  2. AND
  3. NOT
  4. Required/Prohibited (+/-)
  5. Field (:)
  6. Boost (^)
  7. Fuzzy (~) (highest)

Database-Specific Features

PostgreSQL
  • $1, $2 parameterization
  • ~ for regex
  • levenshtein() for fuzzy
  • phraseto_tsquery() for proximity
MySQL
  • ? parameterization
  • REGEXP for regex
  • SOUNDEX() for fuzzy
  • MATCH AGAINST for proximity
SQLite
  • ? parameterization
  • REGEXP for regex
  • FTS5 for proximity
  • json_extract() for JSON
MongoDB
  • BSON query documents
  • $regex for patterns
  • $text for full-text
  • $exists for null checks
